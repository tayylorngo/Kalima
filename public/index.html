    Using Kalima

    Describe a student in plain words and Kalima suggests pre-approved comment codes that fit. Below are tips for getting the best picks.

    How to write a good description

    • Mention strengths and growth areas. Balanced descriptions get balanced suggestions.
    • Be specific — name the subject, grade, and concrete behaviors.
    • Vague descriptions like “good student, works hard” return vague comments.
    • Check the Regents box if the course ends in a Regents exam — #502 will be auto-added.
    • Click “See why” on any card to read why that code was chosen.
    • Don’t love a pick? Click Edit on the description and resubmit — you’ll get a fresh set.

    Example descriptions

    Math

    9th grade Algebra 1. Strong on linear equations and graphing, but rushes word problems and skips showing work.

    ELA

    5th grader. Reads above grade level and joins discussions enthusiastically, but writing lacks structure and revision is rushed.

    Science

    Earth Science, 9th grade. Lab work is careful and accurate, but struggles to explain results in writing. Misses lab make-ups.

    Behavior

    Consistently kind to classmates and helpful in groups, but frequently distracted by phone during independent work.

    Once you have suggestions, click the code badge’s Copy code button to grab the number for your gradebook.

    Before you start

    Kalima sends your description to a Google AI service to suggest comment codes. Please follow these rules so student data stays protected.

    • No names — use “the student” or initials.
    • No student ID numbers or contact info.
    • No medical, family, or disciplinary details that could identify a specific student.
    • Describe behaviors, skills, and habits — not the person.
